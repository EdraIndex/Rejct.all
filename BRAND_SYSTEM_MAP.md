# Brand System Map

How each of the Six Rejections becomes a concrete rule in Content, Product, Drops, and Customer Experience.

**Use this when:** deciding whether a specific idea (a post, a SKU, a drop addition, a reply) belongs in `rejct.all`. Find the column, find the row. If the idea contradicts the cell, it's out.

---

## Master matrix

|   | Content | Product | Drops | Customer Experience |
| - | - | - | - | - |
| **01 · Reject the trend**<br/>*ट्रेंड को नकारो।* | No trending audio. No meme formats. No reactive posts. Grid is dark editorial only — never lifestyle. | No trend-borrowed silhouettes or motifs. Every design decision is Bible-coded, not referenced from what is popular. | The drop exists because the design is right. Never cycle-timed (festive sales, seasonal hype drops). S/S and F/W are calendar markers, not trend cues. | No "ends today" urgency copy. No countdown timers. No `Back in stock soon!` language. |
| **02 · Reject the algorithm**<br/>*एल्गोरिदम को नकारो।* | No hashtags in captions (first comment only if at all). No engagement hooks ("comment YES for..."). No reach-optimised posting times. | No "bestsellers" tagging. No data-driven replicas. No A/B-tested variants for conversion. | No paid-ad-optimised launch calendar. No drop size calibrated to ROAS targets. | No retargeting pixels, no "people also bought," no algorithmic upsell flows. No abandoned-cart guilt emails. |
| **03 · Reject performing**<br/>*दिखावे को नकारो।* | No lifestyle routines, no founder-to-camera reels, no "day in the life." Text-only posts are allowed and encouraged. | Graphic is statement, never costume. Embroidery is quiet (tonal option every drop). The wearer is not dressed up — they are dressed. | No pre-drop hype ladder. Phase 1 (Before) is *coming-soon only*, no product. Build the idea before the object. | No scripted reply templates that sound like a mascot. No "Welcome to the tribe!" onboarding. Replies are direct, not cheerful. |
| **04 · Reject the gaze**<br/>*नज़र को नकारो।* | No "look" content. No "rate my fit." No model-looking-at-lens flirting. Ghost-mannequin, editorial, never influencer-facing. | Back as hero — the brand reveals itself as the wearer walks away, not as they arrive. Script lettering upper back, angel seal, circular ring. | Phase 4 (After) uses real people wearing it, reposted without comment. Never paid creators. No celebrity previews for approval. | No social-proof tickers ("X people viewing this product"). No review-begging emails. No "tag us for a chance to be featured." |
| **05 · Reject the label**<br/>*लेबल को नकारो।* | No `men's` / `women's` language in captions or product pages. No `for him / for her` gift-guides. Indian/Western framing is refused in copy. | XS–XXL, one genderless size system. Cuts do not lean masculine or feminine. No category-forcing names (no `workwear`, no `athleisure`). | No `menswear drop` or `womenswear drop`. No Indian-vs-global split. One drop, one audience. | Checkout + account flows genderless. No "Ms/Mr" prefix fields. Size guide uses body measurements, not stereotypes. |
| **06 · Reject everything that isn't you**<br/>*वो सब नकारो जो तुम नहीं हो।* | Every line passes the voice test (Bible §07). Hindi and English both native. No ironic register, no aspirational register. | Every piece passes the Six Rejections test as a whole. If a sample lands but feels off-position, it is killed — even if it would sell. | Every drop audited before launch: what did we refuse to include? If the answer is "nothing," the drop is probably wrong. | Replies sound like us — not a chatbot, not an intern imitating a brand, not hospitality-script warm. Silence is sometimes the correct reply. |

## How to use this map

1. **You have an idea.** A post, a SKU addition, a collab, a reply.
2. **Pick the column** (Content / Product / Drops / CX).
3. **Read all six cells in that column.** The idea must not contradict any of them.
4. **If unsure about one rejection:** read the Bible §03 for that rejection. The gap in this map is where judgment goes.
5. **If the idea passes, ship it.** If it fails on any rejection, either redesign or kill.

## Worked examples

**Idea:** "Let's run a Diwali bundle — buy any two tees, 15% off."
- Content row 01: cycle-timed → fails Reject the trend.
- Product row 01: unchanged.
- Drops row 01: festive cycle → fails.
- CX row 02: discount + retargeting → fails Reject the algorithm.
- CX row 01: urgency copy → fails.
- **Verdict:** killed across four cells. No.

**Idea:** "Photograph Drop 01 on a friend who just bought RJ-005 and loves it, repost without caption."
- Content row 04: real people, not paid creators → passes Reject the gaze.
- Content row 03: not performing, not a shoot, not a routine → passes.
- CX row 04: not begging, not scripted → passes.
- Drops row 04: Phase 4 (After) model → passes.
- **Verdict:** ship.

**Idea:** "A women's-cut version of RJ-004."
- Product row 05: violates genderless sizing system → fails Reject the label.
- **Verdict:** no. The existing XS–XXL is the answer.

**Idea:** "Post a still of the `REJECT` collar detail with just the caption `नज़र को नकारो।`"
- Content row 02: no hashtag, no reach chase → passes.
- Content row 03: text + detail, not lifestyle → passes.
- Content row 04: ghost-style detail, not gaze-facing → passes.
- **Caveat:** Playbook 02 says both languages in the same caption. The Hindi alone would violate the caption playbook. Add the English line.
- **Verdict:** ship with both languages.

## When this map is wrong

If you encounter a real decision and this map gives no answer — or gives the wrong answer — the map is incomplete. File a decision record in `docs/decisions/` proposing the new cell, and update this map. The Bible stays stable; this map gets sharper over time.
