# Adopt Project Operating System

Date: 2026-04-24
Status: accepted

## Context

Rejct.all was a position in the founder's head with no documented system. Every decision risked drift.

## Decision

Adopt a four-document OS inside the repo:
- `PROJECT_OPERATING_SYSTEM.md`
- `PLAYBOOK_LIBRARY.md`
- `COMMAND_CENTER.md`
- `CLAUDE.md`

Plus scaffolding folders: `second-brain/`, `docs/`, `prompts/`, `templates/`, `output/`, `.claude/skills/`.

## Why

- The position (rejection system, bilingual, genderless, non-performative) is too easy to erode under pressure without a written test.
- Playbooks prevent re-deciding recurring situations (drops, press, collabs).
- A command center concentrates live state in one file.
- Claude can act as an operator only with written tone rules.

Tested against OS pillars: this decision sharpens Pillar 4 (internal alignment over external validation) and makes Pillar 1 (refusal as product) operational.

## Consequences

- All future work routes through the OS files.
- Changes to the OS require a written argument and a 7-day cooling-off period.
- This file becomes the template for future decision records.
